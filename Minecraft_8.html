《指令方块高级教程》

第八章 实体指令

实体指令只能针对实体执行。而所谓实体，就是我的世界中的那些会动的物体，像我家，矿车，箭等（但是，有一些实体指令对于交通工具（例如矿车）、弹射物（例如箭）不起作用），之所以会出现这样的问题，是因为我的世界中不同类型的实体所对应的程序略有不同。每种实体都有不同的能力，因此，它们要比那些静止不动的方块复制得多。例如狼这种实体，它可以是愤怒的（也就是敌对的）；也可以是被驯化的（友好的），你还可以让两只狼在一起生出小狼。而相比之下，一个方块的特性就要少得多。

实体指令包括：

/effect
/entitydata
/kill
/particle
spreadplayers
/summon
/tp

/effect指令

/effect指令可以给予或清除实体某种药水效果（包括玩家）。例如，你可以给所有进入某个昏暗地牢的玩家赋予一个失明的效果，来使他们无法看清远处的东西，即使是他放置了火把也不会有什么帮助。你可以通过附录E来查看所有效果。

格式

/effect <目标实体> <药水效果> [持续时间] [等级] [粒子隐藏]

  • 用玩家名或选择器为参数<目标实体>赋值，他们将在指令执行时获得特定药水效果。
  • 用效果的ID名称为参数<效果>赋值。
  • [持续时间]是可选参数，代表药水效果的持续时间。单位是秒，默认值为30。（当你把这个值设定成大于9999的值后，持续时间将变为无限。）
  • [等级]是可选参数，代表药水效果的强度，默认值是0，范围是从0（最弱，但是有效果）~255（最强）。
  • [粒子隐藏]是可选参数，可以隐藏药水效果产生的粒子，默认值是false（不隐藏），你可以将其改为true（隐藏）。

如果要移除实体的效果，请输入：

/effect <目标实体> clear

例子
/effect @p fire_resistance 120 2 [hideparticles]
/effect @a clear

/entitydata指令

/entitydata指令可以设置实体的数据标签，你可以设置实体的身上盔甲的种类，以及它手中的物品等等。

格式

/entitydata <目标实体> <数据标签>

  • <目标实体>参数的值只能是非玩家实体，因此要用选择器为其赋值。
  • 用目标实体的数据标签以及它们的值为参数<数据标签>赋值。

例子
/entitydata @e[type=Slime] {Motion:[0.0,1.0,0.0],Size:20}

/kill指令

/kill指令非常简单，也非常实用。它经常被用来移除（杀死）一些实体。这些实体包括：玩家，生物，交通工具，弹射物等。

格式

/kill [目标实体]

  • [目标实体]是一个可选参数，它的默认值是指令发出者本人（执行后的效果就是自杀），当然，这可以让你快速回到出生点（不过，你死后掉落的物品就很难找到了。）；你可以用玩家名或选择器为其赋值，它们将会被击杀。

例子
/kill meganfair
/kill @e[type=Zombie]

/particle指令

/particle指令可以在一个区域内制造粒子效果。

格式

/particle <粒子名称> < x y z> < xd yd zd> <速度> [数量] [模式]

  • 用你希望生成粒子所对应的ID名称为参数<粒子名称>
  • 用粒子生成的位置为参数< x y z>赋值
  • 用粒子生成的范围为参数< xd yd zd>赋值，所谓粒子生成的范围，就是以上一个参数< x y z>为中心，生成粒子区域的宽度（延x轴），高度（延y轴）和另一方向上的宽度（延z轴）。例如，当你用1 1 1为它赋值，则粒子会在中心点周围1格内生成；当参数[数量]为设置或是值为0时，它还可以改变某些粒子的颜色。
  • 用粒子运动的速度为参数<速度>赋值，它的值不能小于0。过快的粒子速度会导致粒子存在的时间非常短，以至于你很难看到它们。所以，让粒子运动得慢一点（速度为0.05左右）比较好。
  • [数量]是可选参数，你可以用单位时间内粒子生成的数量为其赋值，大于0（不包括0）。切忌使用过高的数值，则会导致你的游戏卡顿甚至是崩溃。在我的电脑中，从生成大约一万个粒子时开始出现卡顿。
  • [模式]是可选参数，它有两个值，分别是“正常”和“强制”。默认值为“正常”：生成的粒子数量将受到玩家在视频设置中设定的粒子量的限制。反之则不会受限。

例子
/particle happyVillager ~2 ~ ~2 1 1 1 .05 1000
/particle mobSpell ~2 ~1 ~ 2 3 1 1 5000

/summon指令

/summon指令非常有用，你可以通过它来生成我的世界中的任何实体，而且其中一些是你平时无法看到的生物，例如骷髅马。你还可以改变实体的数据标签，例如让一个怪物穿上附魔的盔甲。

格式

/summon <实体名称> [x y z] [数据标签]

  • 用实体ID为参数<实体名称>赋值
  • [x y z]是可选参数，用实体生成的位置为其赋值。
  • [数据标签]是可选参数，用实体的数据标签为其赋值。

例子
生成7个叠在一起的史莱姆，由下向上一个比一个大:
/summon Slime ~ ~ ~ {Size:15,Riding:{id:Slime,Size:13,Riding:{id:Slime,Size:11,Riding:{id:Slime,Size:9,Riding:{id:Slime,Size:7,Riding:{id:Slime,Size:5,Riding:{id:Slime,Size:3}}}}}}}
生成一个用钻石换泥土的村民：
/summon Villager ~1 ~ ~ {Offers:{Recipes:[{buy: {id:dirt,Count:1},sell:{id:diamond,Count:1},rewardExp:false}]}}
生成一个带鞍的僵尸马：
/summon EntityHorse ~0 ~1 ~0 {Type:3,Tame:1,SaddleItem:{id:329}}
生成一个超能僵尸：它有200点生命，以及15点攻击伤害。
/summon Zombie ~0 ~1 ~0 {Attributes:[{Name:generic.maxHealth,Base:200},{Name:generic.attackDamage,Base:15}]}

用/summon指令制作烟花

你可以用/summon指令来制作烟花：

/summon FireworksRocketEntity ~ ~5 ~ {LifeTime:20,FireworksItem:{id:401,Count:1,tag:{Fireworks:{Explosions:[{Type:1,Flicker:1,Trail:1,Colors:[2516601],FadeColors:[3932152]}]}}}}

除了实体名称和xyz坐标外，这个指令中还需要用到几个数据标签：

  • LifeTime：它代表烟花火箭飞行的时间（秒）
  • FireworksItem：“id:401,Count:1”不用改动，但是，其中还有一个标签，来定义这个烟花：
    • Type：代表烟火的形状
    • Flicker（开启：1或关闭：0）：代表烟火是否闪烁。
    • Trail（开启：1或关闭：0）：代表烟火是否有尾迹。
    • Colors：烟火的颜色。（其中可以有多种颜色）
    • diamond trail：烟火的下层颜色。（其中可以有多种颜色）

    不难看出，一个数据标签中（特别是那些一个套一个的）的种种括号错综复杂。而且这还仅仅是烟花中的数据标签，实际上，summon指令中可以使用大量不同的数据标签。你可以使用在线指令生成器来“多快好省”的制造自己的烟花：指令生成器，你还可以通过颜色生成器来选择自己的颜色：颜色生成器

    /tp指令

    /tp指令可以把你或其他人及实体瞬间传送到某个地方，这个地方可以是实体，也可以是某个坐标。例如，你可以用指令方块设置出传送点，并在上面放置一个按钮；这样一来，你就可以将自己传送至某个特定位置。

    格式

    将实体传送到某个玩家或实体

    /tp [被传送者] <目标实体>

    • [被传送者]是可选参数，默认是指令发出者。用玩家名或选择器为其赋值
    • 用传送目标（玩家或实体）为参数<目标实体>赋值。切记，如果你使用选择器的话，传送目标只能有一个，你可能需要在其中加一个“c=1”（参见第三章目标选择器部分）。

    /tp [被传送者] < x y z> [< y-旋转角> < x-旋转角>]

    • [被传送者]的用法和上面相同
    • 用传送的位置（xyz）为参数< x y z>赋值。
    • [< y-旋转角> < x-旋转角>]是可选参数，其中的两个参数必须一起使用。用水平（沿y轴）旋转的角度为< y-旋转角>赋值（值为180时玩家面向正北，为90时向正西，0时向正南，-90时向正东）；用垂直（沿x轴）旋转角度为参数< x-旋转角>赋值（值为90时玩家视角向下，值为-90时玩家视角向上）。

    例子
    /tp 0 64 0
    /tp meganfair megorniuspi
    /tp meganfair ~10 ~ ~10 180 90
